使用ELK(ElasticSearch+Logstash+Kibana)搭建实时日志分析平台

发表于 2019-05-27 | 更新于 2019-06-01

ELK组件套装

ELK 由三部分组成Elasticsearch、Logstash、Kibana, Elasticsearch是一个近似实时的搜索平台,它让你以前所未
有的速度处理大数据成为可能。

  • Logstash：日志收集工具，可以从本地磁盘，网络服务（自己监听端口，接受用户日志），消息队列中收集各种各样的日
    志，然后进行过滤分析，并将日志输出到Elasticsearch中。
  • Elasticsearch：日志分布式存储/搜索工具，原生支持集群功能，可以将指定时间的日志生成一个索引，加快日志查询和访
    问。
  • Kibana：可视化日志Web展示工具，对Elasticsearch中存储的日志进行展示，还可以生成炫丽的仪表盘。

使用ELK可以高效地完成日志的收集和方便地完成关键日志的搜索,解决日志种类多且分散在不同的位置(可能是一个集群环境几十台主机或者多个不同业务的集群)难以统一查询的问题,使得零散的日志记录能够统一的在一个平台上进行收集检索和分析,极大提高了日志搜索和定位的效率。

搭建ELK环境(CentOS6):

  • Jdk环境
  • Elasticsearch-6.4.3 安装

使用非root用户,从5.0开始，ElasticSearch 安全级别提高了，不允许采用root帐号启动

1.解压安装包

tar -xzf elasticsearch-6.4.3.tar.gz

  1. 修改配置

elasticsearch的主配置文件在config/elasticsearch.yml中，该配置文件包含集群、节点、网络和数据存储等等的重要配置。

数据存储位置(单个目录设置)
path.data: /XXX/data

找到network.host这行配置，把注释解开，值设置为0.0.0.0 取消主机限制
network.host: 0.0.0.0

Centos6不支持SecComp，而ES5.2.0默认bootstrap.system_call_filter为true
禁用：在elasticsearch.yml中配置bootstrap.system_call_filter为false，注意要在Memory下面:
bootstrap.memory_lock: false
bootstrap.system_call_filter: false

  1. 启动

cd 到bin目录

./elasticsearch

-d 后台运行

  1. 修改文件句柄

提示
ERROR: [4] bootstrap checks failed … open too many files … 错误时, 修改系统文件句柄

切换到root用户

修改配置sysctl.conf:

$ vi /etc/sysctl.conf

添加下面配置：
vm.max_map_count = 655360

退出并保存，然后执行命令：
$ sysctl -p

设置/etc/security/limits.conf

  • Logstash-6.4.3 安装

非root用户

  1. 解压安装包

tar -xzvf logstash-6.4.3.tar.gz

  1. 创建 file-es.conf 自定义的配置文件

config目录

input {
file {
path=>”/XXX/test.log” #监听输入日志文件
start_position=>”beginning”
}
}

output {
elasticsearch {
hosts=>[“XXX.XXX:9200”] #日志输出到es服务
index=>”apple-index-%{+YYYY.MM.dd}” #指定es的index
}
stdout { codec=>rubydebug }
}

  1. 启动运行

sh bin/logstash -f config/file-es.conf

  • Kibana-6.4.3 安装

  1. tar -xzf kibana-6.4.3-darwin-x86_64.tar.gz

  2. 修改配置

cd config 目录, kibana.yml

kibana服务地址
server.port: 5601
server.host: “XXX”

指定elasticsearch服务
elasticsearch.url: “http://XXX:9200“

  1. 启动

./bin/kibana

  1. 访问

http://10.10.105.170:5601

  1. 配置 kibana 定义索引值

Management –> Create index pattern (配置 index pattern apple-index-*)
