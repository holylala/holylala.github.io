Elasticsearch 在股票搜索中的一种应用

发表于 2019-05-20 | 更新于 2019-05-27

What’s Elasticsearch

官方定义:Elasticsearch 是一个分布式、RESTful 风格的搜索和数据分析引擎，能够解决不断涌现出的各种用例。通过 Elasticsearch，能够执行及合并多种类型的搜索（结构化数据、非结构化数据、地理位置、指标）。通过有限状态转换器实现了用于全文检索的倒排索引，实现了用于存储数值数据和地理位置数据的 BKD 树，以及用于分析的列存储。而且由于每个数据都被编入了索引，可以用快到令人惊叹的速度使用和访问所有数据。

总结有三点:

- 分布式实时文件存储，并将每一个字段都编入索引，使其可以被搜索。
- 实时分析的分布式搜索引擎。
- 可以扩展到上百台服务器，处理PB级别的结构化或非结构化数据。

喜欢炒股的同学都知道,当我们使用股票行情软件购买股票时,除了行情首页提供的一些股票分类信息之外有时候会定向搜索一些股票,加入自选重点关注。通常的做法可以通过用户相对模糊的输入,使用SQL模糊匹配查询数据库中相关股票信息并返回结果这种方式略显笨重,准确性也很难保证。本文重点介绍如何使用 Elasticsearch (基于Java Api)更高效更为简便地完成股票信息的检索。

搭建Elasticsearch环境(单个服务节点,CentOS 6)

  • 安装包下载

通过 https://www.elastic.co/cn/downloads/past-releases 下载 elasticsearch-2.4.5.tar.gz

使用非root用户

解压 tar -xzf elasticsearch-2.4.5.tar.gz 到相关用户目录 比如 /home/XXX/apps

  • 修改相关配置

cd 到配置目录 /home/XXX/apps/elasticsearch-2.4.5/config

修改配置文件 elasticsearch.yml

  • 启动服务

bin目录

./elasticsearch

加 -d 后台运行

  • 修改文件句柄

提示
ERROR: [4] bootstrap checks failed … open too many files … 错误时, 修改系统文件句柄

切换到root用户

修改配置sysctl.conf:

$ vi /etc/sysctl.conf

添加下面配置：
vm.max_map_count = 655360

退出并保存，然后执行命令：
$ sysctl -p

设置/etc/security/limits.conf

  • 访问服务显示服务信息

curl http://服务ip:9200/
